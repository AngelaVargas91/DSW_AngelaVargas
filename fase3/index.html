El código CSS facilita la vida de un desarrollador front-end al separar la estructura de un documento HTML de su presentación. En otras palabras, el HTML actuará como la estructura de la web, definirá su estructura básica y CSS agregará una capa totalmente personalizada en la que la web determinará su apariencia final. Siguiendo esta base, es muy fácil para un diseñador web realizar cambios en la apariencia de un sitio web sin afectar significativamente su contenido. El contenido siempre será el mismo, cambia según lo que podamos ver. CSS es fácil de entender y aprender, y nos brinda un fuerte control sobre cómo se diseñan los documentos HTML.

En la década de 1970, mucho antes de la llegada de la web, se desarrolló un sistema de codificación primitivo llamado SGML. Desde la llegada de SGML, ha sido necesario definir un mecanismo que permita la aplicación consistente de diferentes estilos a los documentos electrónicos. Con la llegada de Internet, esto se hizo cada vez más evidente y fue necesario incorporar una tecnología para poder agregar una capa adicional de personalización a los documentos HTML. La falta de estándares significó que compañías como Microsoft y Netscape, que competían en ese momento por el navegador más popular, convirtieron la web en un lugar donde el mensaje "Este sitio solo es compatible con Internet Explorer" es casi la norma. Era casi imposible crear un sitio web con el mismo aspecto en diferentes navegadores en ese momento.
